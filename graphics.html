Graphic elements

Three signature gestures define D8A's visual language: the infinity loop with its signal pulse, line art, and — new in v3.0 — the redline: the red stroke that underlines the point or strikes the rejected.

The infinity symbol

The infinity symbol (∞) represents continuous data delivery — the never-ending loop of request, spec, build, ship. It's our primary graphic mark alongside the wordmark.

The signal pulse

The pulse is a short Redline segment (~15% of the path) traveling the loop — data moving through request → spec → build → ship. In motion it's the loading indicator and the hero animation. Static, it's a single red segment on a gray loop.

Animated pulse + tail · loading & hero
Static pulse · print & stills
  • One pulse per composition. The pulse counts as the surface's one red statement.
  • The loop stays gray (Gray 700 #404040 on dark) — only the pulse is red. Never draw the full loop in red at large scale.
  • The tail is atmosphere: a wider, 25%-opacity trail behind the pulse (Red Fade). It softens the hard edge between gray and red — use it whenever the pulse animates.
  • Motion spec: 3s per lap, linear, infinite. No easing — pipelines don't hesitate.
  • On light canvas the pulse uses Red 600 (#e4002b) on a Gray 300 loop.

Infinity usage

  • Hero backgrounds: Use at large scale (200px+), low opacity (4–8%), as a subtle watermark.
  • Section dividers: Use inline at medium scale with thin stroke as a visual break.
  • Loading states: Animated infinity path as a loading indicator.
  • Favicons & app icons: Simplified thick-stroke version.
  • Never fill the infinity. It's always a stroke/outline. The openness is intentional — it represents flow.

The redline

The brand's bluntest gesture: a straight red stroke, drawn like an engineer's pen on a spec. It does exactly two things — it strikes what we reject, and it underlines what we mean. Nothing else.

Ship data, not excuses. The strike · reject the noise
Shipped 3× faster. The underline · mark the point
  • Straight and honest. No hand-drawn wobble, no marker texture, no swooshes. The redline is set with type tools: text-decoration-thickness: 0.09em, offset 0.14em for underlines.
  • Strike sparingly. One struck word or phrase per surface, and only words we genuinely reject: excuses, tickets, handoffs, lock-in. Never strike people, brands, or competitors by name.
  • The redline counts as the surface's one red statement. A strike and a red CTA on the same slide is one red too many.
  • Color: Redline (#ff2b36) on dark, Red 600 (#e4002b) on light. The struck text itself keeps its normal color — only the line is red.

Line art

D8A's secondary graphic language is line art — thin, precise, geometric lines that suggest data flow, connections, and architecture. Inspired by the blueprint/canvas view in the product.

Node & connection pattern
Grid & flow · red = shipped

Line art rules

  • Stroke weight: 1–2px for detail, 0.5px for grid backgrounds.
  • Color: Gray scale carries the structure. Black (#0a0a0a) for primary strokes, Gray 400 (#a3a3a3) for secondary, Gray 200 (#e5e5e5) for background grids. The active path — the one that shipped — may be Redline. Max one red path per composition.
  • Nodes: Always circles, never squares. Stroke only (no fill) for primary nodes. Small filled dots for waypoints.
  • Flow direction: Left to right, or top to bottom. Never right-to-left.
  • No decorative curves. Lines are straight or use 90-degree angles. Bezier curves only for the infinity symbol.

Do / Don't

Do
Use line art to illustrate data flow, architecture, and connections between components
Don't
Use line art as pure decoration with no informational purpose
Do
Keep line art minimal — fewer elements with clear relationships
Don't
Create busy, tangled diagrams with dozens of overlapping lines
