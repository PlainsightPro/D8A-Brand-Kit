Graphic elements

Three signature gestures define D8A's visual language: the infinity loop with its prism signal pulse, line art, and the prism mark — the cobalt gradient that lifts the accent word or underlines the point.

The infinity symbol

The infinity symbol (∞) represents continuous data delivery — the never-ending loop of request, spec, build, ship. It's our primary graphic mark alongside the wordmark.

The signal pulse

The pulse is a short prism segment (~15% of the path) traveling the loop — data moving through request → spec → build → ship, refracting from indigo to blue as it goes. In motion it's the loading indicator and the hero animation. Static, it's a single gradient segment on a grey loop.

Animated pulse + tail · on light
Animated pulse · on ink
  • One pulse per composition. The pulse counts as the surface's one prism statement.
  • The loop stays grey (Line #d4d4d4 on light, #3a3550 on ink) — only the pulse carries the gradient. Never draw the full loop in the gradient at large scale.
  • The tail is atmosphere: a wider, 28%-opacity trail behind the pulse (Prism Fade). It softens the edge where grey meets gradient — use it whenever the pulse animates.
  • Motion spec: 3s per lap, linear, infinite. No easing — pipelines don't hesitate.
  • Gradient direction follows the path, indigo leading into blue — the same 110° refraction as everywhere else.

Infinity usage

  • Hero backgrounds: Use at large scale (200px+), low opacity (4–8%), as a subtle watermark.
  • Section dividers: Use inline at medium scale with thin stroke as a visual break.
  • Loading states: Animated infinity path as a loading indicator.
  • Favicons & app icons: Simplified thick-stroke version.
  • Never fill the infinity. It's always a stroke/outline. The openness is intentional — it represents flow.

The prism mark

The brand's emphasis gesture: the cobalt gradient applied to the one thing that matters in a line of type. It does exactly two things — it lifts the accent word into the gradient, and it underlines the point with a gradient rule. Nothing else.

Ship data, not excuses. The prism word · lift the accent
Shipped 3× faster. The prism underline · mark the point
  • One word, or one underline — not both. The prism mark is the surface's one statement. A gradient word and a gradient CTA on the same slide is one prism too many.
  • Lift what we affirm. The gradient goes on the word we mean — data, shipped, faster — not on what we reject. This world persuades by lighting the point, not striking the enemy.
  • Straight and precise. The underline is set with type tools: a gradient rule at 0.14em, offset just below the baseline. No wobble, no marker texture.
  • Large only. The prism word works at 24px+ where the gradient reads; below that, use solid Cobalt 600.

Line art

D8A's secondary graphic language is line art — thin, precise, geometric lines that suggest data flow, connections, and architecture. Inspired by the blueprint/canvas view in the product.

Node & connection pattern
Grid & flow · prism = shipped

Line art rules

  • Stroke weight: 1–2px for detail, 0.5px for grid backgrounds.
  • Color: Grey scale carries the structure. Ink (#15121f) for primary strokes, Mauve (#6c6781) for secondary, Line (#e7e3f2) for background grids. The active path — the one that shipped — may carry the prism gradient. Max one gradient path per composition.
  • Nodes: Always circles, never squares. Stroke only (no fill) for primary nodes. Small filled dots for waypoints.
  • Flow direction: Left to right, or top to bottom. Never right-to-left.
  • No decorative curves. Lines are straight or use 90-degree angles. Bezier curves only for the infinity symbol.

Do / Don't

Do
Use line art to illustrate data flow, architecture, and connections between components
Don't
Use line art as pure decoration with no informational purpose
Do
Keep line art minimal — fewer elements with clear relationships
Don't
Create busy, tangled diagrams with dozens of overlapping lines
