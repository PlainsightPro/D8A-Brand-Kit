How to use this kit

The rules that apply to every D8A deliverable. If you're creating anything with our name on it, start here.

The six rules

01

Ship it clean

Every deliverable uses the correct logo, colors, and fonts. No exceptions. If the template exists, use it.

02

No fluff

If a sentence doesn't say anything, delete it. D8A copy is direct, specific, and useful. Buzzwords are banned.

03

Flat, not fancy

No shadows, no gradients, no gloss. Surfaces are separated by borders, not depth effects. Let the content breathe.

04

Monochrome first

Design in black and gray first. If it doesn't work without color, it doesn't work. Color is emphasis, not decoration.

05

Consistency over creativity

Every D8A deliverable should look like it came from the same team. Reuse patterns. Follow the grid. Don't reinvent.

06

Prove it

Don't claim — show. Every bold statement should have a number, a screenshot, or a customer quote behind it. D8A earns trust by demonstrating, not declaring.

Pre-send checklist

Logo is the correct variant for this background?
All text is in Inter?
No shadows, gradients, or glossy effects anywhere?
Copy is direct and free of buzzwords?
Color usage follows the monochrome palette?
Every claim is backed by evidence?

When in doubt

Open this brand kit. If what you're making doesn't look like it belongs next to these pages, revisit. The canonical reference is always d8a.pro and this kit.
